Exercise 1: Working with AWS EC2

Amazon Elastic Compute Cloud (EC2) is for running computations on a cloud server.

  1. Launch a new EC2 instance with default settings (Amazon Linux 2023 AMI, t2.micro).

    • Make sure to select vockey for your ssh keys

  2. Get the public DNS of the server, and SSH into it using your PEM file

$ ssh -i "vockey.pem" ec2-user@INSTANCE_PUBLIC_IP
  1. In a separate local shell, securely upload your files in with scp
$ scp -i "vockey.pem" -r ex1.py ec2-user@INSTANCE_PUBLIC_IP:.
  1. Run the example file in Python on EC2
[AWS] python3 ex1.py
  1. Securely download your files to local machine with
$ scp -i "vockey.pem" -r ec2-user@INSTANCE_PUBLIC_IP:results.txt .

Note: The syntax for scp is

scp -i "NAME_OF_PEM_FILE.pem" ORIGIN_DESTINATION TARGET_DESTINATION

The -r flag indicates to transfer recursively, i.e. an entire directory.

IMPORTANT: TERMINATE THE EC2 INSTANCE WHEN DONE!!!

Exercise 2: Working with AWS Lambda Functions

AWS Lambda is a "serverless" service which is designed to easily scale, typically meant for running many quick computations in response to some event.

In this exercise, you will use Lambda functions to generate a list of prime numbers up to a specified number N. The provided code takes as input a number n and determines whether it is a prime number:

def is_prime(n):
    """
    Check if n is a prime number.
    """
    if n <= 1:
        return False
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            return False
    return True

Recall that your Lambda functions must include lambda_handler:

def lambda_handler(event, context):
    results = 'Hello world'
    return {
        'statusCode': 200,
        'body': json.dumps(results)
    }

The inputs to the Lambda functions are in event, which is a JSON object such as

{
    'N' : 200,
    'other' : 'stuff'
}

Exercise 2b: Working with boto3

For reproducibility or collaboration, it is a good idea to do it programmatically. You can do this with Amazon's SDK called boto3.

In this part, repeat the exercise above but programmatically. A sketch of the code is as follows:

import boto3
import json

aws_lambda = boto3.client('lambda')
iam_client = boto3.client('iam')
role = iam_client.get_role(RoleName='LabRole')

# Open the zip file with your Lambda function
with open('<lambda_func_zip>.zip', 'rb') as f:
    lambda_zip = f.read()

try:
    # Create a new Lambda function in AWS
    response = aws_lambda.create_function(
        FunctionName='<lambda_function_name>',
        Runtime='python3.11',
        Role=role['Role']['Arn'],
        Handler='<file_name>.<lambda_function_name>',
        Code=dict(ZipFile=lambda_zip),
        Timeout=300
    )
except aws_lambda.exceptions.ResourceConflictException:
    # If it already exists, update it based on the new code
    response = aws_lambda.update_function_code(
        FunctionName='<lambda_function_name>',
        ZipFile=lambda_zip
        )

lambda_arn = response['FunctionArn']
lambda_arn

# Define the inputs/`event` as JSON
event = json.dumps({
    'N' : 200,
    'other' : 'stuff'
})

# Trigger the Lambda function
r = aws_lambda.invoke(FunctionName='<lambda_function_name>',
                      InvocationType='RequestResponse',
                      Payload=event)
json.loads(r['Payload'].read())

When the Lambda function is triggered, you can intuitively think of it as AWS spinning up a new EC2 instance, running the code, then stopping.

Exercise 3: Step Functions

We can execute the Lambda functions above sequentially, for example:

res = []
for n in range(10):
    r = aws_lambda.invoke(FunctionName='<lambda_function_name>',
                      InvocationType='RequestResponse',
                      Payload=event)
    res.append( json.loads(r['Payload'].read()) )

AWS Step Functions are for orchestrating more complicated workflows. A simple example is to run a Lambda function multiple times in parallel.
